# Правила расстановки и фаз — `MatchEngine`

Техническая фиксация того, как сервер держит позиции и фазы относительно правил большого тенниса. Детали геометрии — `geometry.ts`, константы — `constants.ts`.

## Координаты

- **left** — южная половина (`y > NET_Y`), базовая линия у `y = COURT_L`.
- **right** — северная половина (`y < NET_Y`), базовая линия у `y = 0`.
- Подача в корректный **диагональный** сервис-бокс — `inDiagonalServiceTarget` (дежурство / нечёт по ширине).

## Между очками

- Фазы **`point_pause`** и **`sides_change`**: игроки **не интегрируются** по полю — обнуляется ввод (`dx, dy`), позиции не меняются. Иначе после розыгрыша персонажи «доезжают» на старом направлении к сетке до следующей подачи.
- При начале очка **`beginAfterPause`**: оба на **своих базовых линиях** (`baselinePosition`).

## Подача (`serve_ready` | `serve_power` | `serve_aim`)

- **`setMove`** не принимается ни для кого — нельзя «набрать» скорость до удара.
- Каждый тик **оба** игрока жёстко ставятся на свои `baselinePosition` — подающий не уезжает вперёд, приёмник не подходит к сетке (в т.ч. бот не гонится за мячом: см. `BotMatchController.updateBotMove` вне `playing`).

## Розыгрыш (`rally`)

- Движение по половинам — `clampPlayerToHalf`.
- Мяч: гравитация, сетка, отскоки, `serveInPlay` до первого валидного отскока в зоне приёма.

## Таймаут индикатора подачи

- Нет **поражения в матче** за два подброса подряд без удара.
- Таймаут по-прежнему увеличивает счётчик ошибок подачи; при второй ошибке на этом же розыгрыше очка — очко сопернику (`Таймаут подачи`), как при двойной ошибке по таймауту.
