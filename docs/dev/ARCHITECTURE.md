# ARCHITECTURE — Tennis 1v1

## Контекст

Первая версия продукта рассчитана на небольшую аудиторию (до 50 одновременных игроков) с возможностью масштабирования в будущем. Всё разворачивается на собственном сервере. Решения выбраны с учётом надёжности хранения данных, безопасности и комфортного геймплея при низкой латентности.

---

## План

### Стек технологий

| Слой | Технология | Назначение |
|---|---|---|
| Фронтенд | Phaser 3 + TypeScript + Vite | Игровой движок, рендеринг, сборка |
| Бэкенд | Node.js + TypeScript + Socket.io | Реалтайм WebSocket, игровой цикл |
| HTTP API | Express | Авторизация, профиль, история матчей |
| База данных | PostgreSQL | Хранение пользователей, матчей, статистики |
| ORM | Prisma | Типобезопасные запросы, миграции схемы |
| Аутентификация | Firebase Auth | Email + телефон, бесплатно до 10k SMS/мес |
| Токены | JWT | Stateless авторизация между клиентом и API |
| Reverse proxy | Nginx | SSL, раздача статики, проксирование WebSocket |
| Контейнеризация | Docker + docker-compose | Изоляция сервисов, воспроизводимость, откат |
| SSL | Let's Encrypt (Certbot) | Бесплатный HTTPS |

### Управление и мониторинг

| Сервис | Инструмент | Доступ |
|---|---|---|
| PostgreSQL | pgAdmin 4 | Веб-интерфейс в отдельном контейнере |
| Docker контейнеры | Portainer | Веб-интерфейс: логи, перезапуск, статус |
| Метрики сервера | Netdata | CPU, RAM, сеть в реальном времени |
| Firebase Auth | Firebase Console | Веб-панель Google |

Все инструменты управления доступны по защищённым URL на своём домене (например `pgadmin.домен.ru`, `portainer.домен.ru`).

### Структура сервисов (docker-compose)

```
tennis1v1/
├── client/          # Phaser 3 + Vite → статика, раздаётся Nginx
├── server/          # Node.js + Express + Socket.io
├── postgres/        # PostgreSQL
├── pgadmin/         # pgAdmin 4
├── portainer/       # Portainer
├── netdata/         # Netdata
└── nginx/           # Reverse proxy + SSL
```

### Архитектура игрового цикла

- Физика мяча, счёт и состояние матча считаются **на сервере** (server-authoritative)
- Клиент отправляет только пользовательские inputs: направление движения, нажатия кнопок
- Сервер рассылает состояние игры всем участникам комнаты ~60 раз в секунду
- Клиент только рендерит полученное состояние — исключает рассинхронизацию и читерство

### Ассеты

**Источники (бесплатные, лицензия разрешает коммерческое использование):**
- **easyTennis (OpenGameArt, CC-BY 3.0)** — персонажи, корт, мяч. Кредит: Marco Giorgini
- **Kenney Sports Pack + Sound Packs (CC0)** — UI иконки, дополнительные элементы, звуки

**Структура ассетов:**
```
client/public/assets/
├── players/        # спрайт-листы персонажей
├── courts/         # тайлсеты кортов (покрытия: хард, грунт, трава)
├── rackets/        # спрайты ракеток
├── ball/           # мяч (размеры, тень)
├── ui/             # иконки, индикаторы, реакции
└── sounds/         # звуки ударов, событий, UI
```

**Конфигурация ассетов — `assets.config.ts`:**
- Все персонажи, корты, ракетки регистрируются в едином конфиге
- Добавление нового ассета = добавить запись в конфиг, не трогая игровую логику
- Это обеспечивает гибкость: новые стадионы, скины, ракетки добавляются без изменений кода

### Масштабирование в будущем

- **Redis Adapter** для Socket.io — позволяет запускать несколько Node.js процессов за балансировщиком
- **PostgreSQL read replicas** — для разгрузки аналитических запросов администратора
- **CDN** поверх Nginx — для раздачи статики клиента глобально
- **docker-compose → docker swarm / Kubernetes** — без переписывания кода
- **Новые персонажи, корты, ракетки** — добавление через `assets.config.ts` без изменений игровой логики
- **Скины** — перекраска существующих спрайтов, отдельный слой поверх базового персонажа

### Безопасность

- HTTPS обязателен, HTTP редиректит на HTTPS
- JWT токены с коротким сроком жизни + refresh tokens
- Firebase Auth верифицирует SMS и email на стороне Google
- Клиент не доверяется в игровых решениях — вся логика на сервере
- Инструменты управления (pgAdmin, Portainer) закрыты за базовой HTTP авторизацией Nginx и доступны только с доверенных IP

### Защита от спама комнатами

- Один пользователь не может иметь более **3 активных комнат** одновременно (по аккаунту для зарегистрированных, по IP для гостей)
- Не более **10 комнат за 10 минут** с одного IP
- Комната без активности (никто не подключился) автоматически закрывается через **5 минут**
